The [draft v1 specification](https://github.com/tylergannon/go-gen-jsonschema/blob/main/docs/spec/v1.md)
records the original syntax-scanner, schema-rendering, registration, and
generated-code design. It predates `Optional[T]`, `Nullable[T]`, and direct
slices of registered interfaces, so it is not a complete reference for the
current release.

Use these pages and their linked compiling examples for current behavior:

- [Getting started](/getting-started/) for installation and the two build-tagged files.
- [Optional and nullable](/features/optionality/) for required, omitted, and null properties.
- [Enums](/features/enums/) and [Interfaces](/features/interfaces/) for field-level registrations.
- [Provider schemas](/features/providers/) for runtime schema customization.
- [Validation and CI](/guides/validation-ci/) for JSON validation and drift checks.

Use the draft specification as design background when changing the generator,
then verify the current source, examples, generated output, and tests before
relying on a subtle implementation guarantee.
