The [canonical v1 specification](https://github.com/tylergannon/go-gen-jsonschema/blob/main/docs/spec/v1.md)
defines the generator's syntax scanner, schema rendering, registration, and
generated-code contracts.

For day-to-day use, start with these task-focused pages:

- [Getting started](/getting-started/) for installation and the two build-tagged files.
- [Optional and nullable](/features/optionality/) for required, omitted, and null properties.
- [Enums](/features/enums/) and [Interfaces](/features/interfaces/) for field-level registrations.
- [Provider schemas](/features/providers/) for runtime schema customization.
- [Validation and CI](/guides/validation-ci/) for JSON validation and drift checks.

Use the canonical specification when changing generator behavior or checking a
subtle implementation guarantee.
